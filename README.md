## 🆕 Изменения в форке

### 📦 Добавлен пакет `teleop_jetbot` (версия 1.0.1)

Разработан новый ROS2 пакет для упрощенного телеуправления роботом JetBot с клавиатуры через локальную сеть.

**Основные возможности:**
- ⌨️ Управление роботом с клавиатуры компьютера (WASD + QE)
- 🌐 Работа через ROS2 топики по локальной сети
- 🔄 Практическая совместимость Galactic ↔ Humble при одинаковом DDS/RMW и корректном discovery
- 📡 Автоматическая конвертация команд в протокол ESP32
- 🛡️ Безопасность: автоостановка робота при отсутствии команд (5 сек)
- 🚀 Минимальные зависимости - использует существующий `serial_bridge_package`
- 🔧 Надежная обработка ошибок терминала и исключений

**Файлы пакета:**
```
src/teleop_jetbot/
├── launch/
│   ├── teleop_pc.launch.py      # Launch для ПК
│   └── teleop_robot.launch.py   # Launch для робота
├── teleop_jetbot/
│   └── teleop_keyboard.py       # Узел управления клавиатурой
├── README.md                     # Документация пакета
└── LICENSE
```

**Быстрый старт:**

*На роботе (JetBot):*
```bash
cd ~/jetbot_ws
source install/setup.bash
ros2 launch teleop_jetbot teleop_robot.launch.py
```

*На ПК:*
```bash
cd ~/jetbot_ws
source install/setup.bash
ros2 run teleop_jetbot teleop_keyboard
```

**Полная документация:** См. [teleop_jetbot/README.md](teleop_jetbot/README.md)

### 🔧 Технические улучшения

- ✅ Исправлена проблема `termios.error` при запуске через `ros2 launch`
- ✅ Добавлена проверка интерактивного терминала (`isatty()`)
- ✅ Улучшена обработка исключений для стабильной работы
- ✅ Оптимизирован вывод в терминал (без переполнения экрана)
- ✅ Добавлена полная совместимость с WSL (Windows Subsystem for Linux)

### 📋 Архитектура решения

**Разделение ответственности:**
- **ПК (Humble)**: Только узел телеуправления с клавиатуры → публикация в `/cmd_vel`
- **Робот (Galactic)**: Прием `/cmd_vel` → конвертация через `serial_bridge_package` → отправка в ESP32

**Преимущества:**
- 🔌 Модульность: легко заменить источник команд (джойстик, автопилот)
- 🌍 Работа по сети без дополнительных настроек (ROS2 DDS)
- 🔄 Повторное использование существующего кода

**Изменения в структуре проекта:**
- ✨ Добавлен новый пакет в `src/teleop_jetbot/`
- 🔗 Пакет использует существующий `serial_bridge_package` для связи с ESP32
- ⚙️ Совместим со всей существующей инфраструктурой проекта
- 📚 Полная документация в [teleop_jetbot/README.md](teleop_jetbot/README.md)
- 🧪 Протестировано на реальном роботе JetBot с Ubuntu 20.04 (Galactic) и ПК с WSL Ubuntu 22.04 (Humble)

## Предварительные требования

Перед началом работы с проектом, пожалуйста, убедитесь, что ваше окружение соответствует следующим требованиям:

*   **Операционная система:** Для рабочей станции (хоста) рекомендуется **Ubuntu 22.04** нативная или виртуальная.
*   **Фреймворк:** Проект разработан для **ROS2 Humble**. (На самом роботе (Jetson Nano) установлен **ROS2 Galactic** под Ubuntu 20.04.)
*   **Уровень подготовки:** Данная документация и инструкции предполагают, что вы уже знакомы с основами ROS2. Понимание таких понятий, как ноды, топики, сервисы и работа с ними через терминал, является обязательным.

Если вы только начинаете знакомство с ROS2, рекомендуем пройти вводный курс, чтобы чувствовать себя увереннее:
*   **[Бесплатный курс: ROS2 — Введение в робототехнику](https://stepik.org/course/221157/promo)** — этот курс на русском языке покрывает необходимый минимум для понимания дальнейших шагов.

## 1. Подготовка к работе (Setup)

Этот раздел содержит инструкции, которые необходимо выполнить **один раз** для подготовки оборудования и программного обеспечения.

*   **[A00] Проверка подключения (важно!)**
    *   **Описание:** Перед включением робота убедитесь, что все компоненты подключены верно. Неправильное подключение может привести к неработоспособности или поломке.
    *   **Монтажная схема:** [🔌 Монтажная_схема_jetbot_mirea.png](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Схемы/Монтажная_схема_jetbot_mirea.png)

*   **[A0] Подключение к Jetson Nano**
    *   **Описание:** Способы получения доступа к бортовому компьютеру робота (SSH или напрямую).
    *   **Инструкция:** [📄 A0_Подключение_к_Jetson_Nano.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/A0_Подключение_к_Jetson_Nano.pdf)

*   **[A1] Развертывание репозитория с ПО**
    *   **Описание:** Клонирование и сборка рабочего пространства ROS2 на вашем компьютере или на Jetson Nano.
    *   **Инструкция:** [📄 A1_Развёртывание_репозитория_с_ПО_для_jetbot_mirea.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/A1_Развёртывание_репозитория_с_ПО_для_jetbot_mirea.pdf)

*   **[A1.5] ДОП. Настройка OLED-дисплея (статус бар)**
    *   **Описание:** Установка сервиса, который будет выводить на дисплей IP-адрес, пароль, состояние батареи (ток, напряжение, заряд). Выполняется **один раз** после подключения к роботу и клонирования репозитория.
    *   **Инструкция:** [📄 A1_5_Настройка_OLED_дисплея.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/A1_5_Настройка_OLED_дисплея.pdf)

    *   **Инструкция:** [📄 A3_Настройка_симлинка_лидара_RPLidar.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/A3_симлинка_лидара_RPLidar.pdf)

## 2. Работа с реальным роботом

Здесь собраны инструкции для запуска и управления физическим jetbot_mirea.

*   **[R1] Запуск драйверов сенсоров и моторов**
    *   **Цель:** Оживить робота, чтобы он начал публиковать данные с лидара, камер, IMU и принимать команды на движение.
    *   **Инструкция:** [📄 R1_Запуск_датчиков_и_приводного_уровня.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/R1_Как_запустить_датчики_и_приводной_уровень_на_роботе_jetbot_mirea.pdf)
    *   *Необходимые шаги:* [A0], [A1], [A2.2], [A3].

*   **[R2] Управление с клавиатуры**
    *   **Цель:** Заставить робота ехать, используя клавиатуру.
    *   **Инструкция:** [📄 R2_Управление_движением_jetbot_mirea_с_помощью_клавиатуры.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/R2_Управление_движением_jetbot_mirea_с_помощью_клавиатуры.pdf)
    *   *Необходимые шаги:* Сначала выполните [R1].

*   **[R2.1] 🆕 Упрощенное телеуправление с клавиатуры (новый пакет)**
    *   **Цель:** Управлять роботом с клавиатуры ПК через сеть без сложной настройки.
    *   **Инструкция:** См. [teleop_jetbot/README.md](teleop_jetbot/README.md)
    *   **Особенности:** 
        - Простой запуск одной командой
        - Работает через ROS2 топики по локальной сети
        - Совместим с ROS2 Humble на ПК и Galactic на роботе (при одинаковом RMW/DDS и настроенном discovery)
    *   *Необходимые шаги:* Выполните [R1] на роботе.

*   **[R3] Настройка ПИД-регуляторов моторов**
    *   **Цель:** Оптимизировать управление скоростью моторов для более точных движений.
    *   **Инструкция:** [📄 R3_Настройка_ПИД_регуляторов.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/R3_Как_настроить_ПИД_регуляторы.pdf)
    *   *Необходимые шаги:* [A1], [A2.1], [A2.2].

*   **[R4] Картографирование помещения (SLAM) *(в разработке)***
    *   **Цель:** Создать карту помещения, перемещая робота с помощью клавиатуры.
    *   **Инструкция:** [📄 R4_Картографирование_реального_робота.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/R4_Как_запустить_режим_картографирования_для_реального_робота.pdf)
    *   *Необходимые шаги:* Выполните [R1] на роботе, затем следуйте шагам на хосте из инструкции.

*   **[R5] Автономная навигация по карте *(в разработке)***
    *   **Цель:** Загрузить созданную карту и отправить робота в заданную точку, чтобы он двигался туда самостоятельно.
    *   **Инструкция:** [📄 R5_Навигация_реального_робота.pdf](https://github.com/cyberbanana777/mirea_jetbot_full/blob/main/docs/Короткие_инструкции/pdf/R5_Как_запустить_режим_автономной_навигации_по_карте_для_реального_робота.pdf)
    *   *Необходимые шаги:* Сначала выполните [R4] и сохраните карту. Затем следуйте инструкции [R5].
