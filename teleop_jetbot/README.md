# Teleop JetBot

Пакет телеуправления для робота JetBot с клавиатуры через локальную сеть.

## 📋 Описание

Пакет предоставляет простое телеуправление роботом JetBot с клавиатуры компьютера. Команды передаются по локальной сети через ROS2 топики и поступают на робот, где конвертируются в протокол ESP32.

### Архитектура

```
┌─────────────────┐         ROS2 Network        ┌──────────────────┐
│  PC (Humble)    │ ───────────────────────────> │ Robot (Galactic) │
│                 │      /cmd_vel topic          │                  │
│  teleop_pc.     │                              │  teleop_robot.   │
│  launch.py      │                              │  launch.py       │
│                 │                              │                  │
│  ┌────────────┐ │                              │ ┌──────────────┐ │
│  │ Keyboard   │ │                              │ │ Serial Bridge│ │
│  │ Node       │ │                              │ │ Package      │ │
│  └────────────┘ │                              │ └──────┬───────┘ │
└─────────────────┘                              └────────┼─────────┘
                                                          │
                                                  ┌───────▼───────┐
                                                  │  ESP32        │
                                                  │  (firmware)   │
                                                  └───────────────┘
```

## 🚀 Быстрый старт

### Терминал 1: На роботе (JetBot)

```bash
cd ~/jetbot_ws
source install/setup.bash
ros2 launch teleop_jetbot teleop_robot.launch.py
```

### Терминал 2: На компьютере (ПК) - ИНТЕРАКТИВНЫЙ

```bash
cd ~/jetbot_ws
source install/setup.bash

# ⚠️ ВАЖНО: Запустите ТЗ ПРЯМОЙ КОМАНДОЙ (не через launch):
ros2 run teleop_jetbot teleop_keyboard
```

После запуска вы увидите подсказку по управлению и сможете управлять роботом с клавиатуры.

## ⌨️ Управление

По умолчанию узел показывает привет сообщение с названием команд:

```
========== JETBOT KEYBOARD TELEOPERATION ==========
Movement:
  W - forward        S - backward
  A - left           D - right
Rotation (in place):
  Q - left           E - right
Control:
  X or Space - stop  Ctrl+C - exit
====================================================
```

После этого отображается текущая скорость одной строкой:

```
Linear: 0.500 m/s | Angular: 1.500 rad/s
```

| Клавиша | Действие |
|---------|----------|
| **W** | Движение вперёд |
| **S** | Движение назад |
| **A** | Движение влево |
| **D** | Движение вправо |
| **Q** | Поворот влево на месте |
| **E** | Поворот вправо на месте |
| **X** или **Пробел** | Стоп |
| **Ctrl+C** | Выход |

### Особенности управления

- Скорость изменяется плавно с каждым нажатием клавиши
- Максимальная линейная скорость: **0.5 м/с**
- Максимальная угловая скорость: **1.5 рад/с**
- Автоматическая остановка при выходе из программы

## 📦 Зависимости

### На роботе (обязательно)
- `serial_bridge_package` - для связи с ESP32
- ESP32 с прошивкой из `esp_firmware/esp32_firmware/`

### На ПК
Нет специальных зависимостей - узел работает в текущем терминале

## 🔧 Протокол связи

### Формат команд для ESP32

Команды передаются в формате:
```
$1;<linear_velocity>;<angular_velocity>;#
```

Пример:
```
$1;0.300;0.500;#  # Движение вперёд с поворотом влево
$1;0.000;0.000;#  # Остановка
```

### ROS2 топики

| Топик | Тип | Описание |
|-------|-----|----------|
| `/cmd_vel` | `geometry_msgs/Twist` | Команды скорости от ПК к роботу |
| `/low_level/serial/cmd` | `std_msgs/String` | Команды в формате ESP32 |
| `/low_level/serial/feedback` | `std_msgs/String` | Обратная связь от ESP32 |
| `/odom` | `nav_msgs/Odometry` | Одометрия от ESP32 |

## 🌐 Настройка сети

Для работы через локальную сеть необходимо настроить ROS2 discovery между ПК и роботом.

### На обоих устройствах

Добавьте в `~/.bashrc`:

```bash
# Для локальной сети (замените на ваш диапазон IP)
export ROS_DOMAIN_ID=42  # Единый домен для ПК и робота

# Опционально: указание IP интерфейса
export ROS_LOCALHOST_ONLY=0
```

Применить изменения:
```bash
source ~/.bashrc
```

### Проверка соединения

На роботе:
```bash
ros2 topic list
```

На ПК (после запуска teleop_robot.launch.py на роботе):
```bash
ros2 topic list
# Должен появиться топик /cmd_vel
```

## 🛠️ Решение проблем

### Проблема: Команды не доходят до робота

**Решение:**
1. Проверьте, что `ROS_DOMAIN_ID` одинаковый на обоих машинах
2. Проверьте сетевое подключение: `ping <IP_робота>`
3. Убедитесь, что файрволл не блокирует ROS2 (порты UDP multicast)

### Проблема: ESP32 не получает команды

**Решение:**
1. Проверьте serial порт: `ls -l /dev/esp32`
2. Проверьте права доступа: `sudo chmod 666 /dev/ttyUSB0` (или ваш порт)
3. Проверьте топик: `ros2 topic echo /low_level/serial/cmd`

### Проблема: Узел не реагирует на клавиатуру

**Решение:**
1. **Используйте прямой запуск:** `ros2 run teleop_jetbot teleop_keyboard`
   - Это гарантирует, что stdin будет интерактивным терминалом
2. Убедитесь, что терминал находится в фокусе
3. Если используете launch файл, узел выдаст warning о недоступности интерактивного ввода

### Проблема: `termios.error: Inappropriate ioctl for device`

**Решение:**
- Эта ошибка возникает при запуске через `ros2 launch` без интерактивного терминала
- Используйте: `ros2 run teleop_jetbot teleop_keyboard`

## 📝 Примечания

- **Совместимость**: Пакет работает с ROS2 Galactic (робот) и Humble (ПК)
- **Безопасность**: Робот автоматически останавливается через 5 секунд без команд (защита на уровне ESP32)
- **Производительность**: Команды публикуются с частотой ~10 Гц
